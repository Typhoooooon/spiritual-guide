知识架构（修正版）

预设角色 + 后人解读 + 在线检索 + 自由输入自动研究

整体数据流

预设角色（库内） → 本地配置文件 + LLM 知识
自由输入（库外） → 在线搜索 → 提取资料 → 构建临时角色 → LLM 生成

1. 预设角色升级：原文 + 后人解读

每个预设思想家的配置文件扩展为两层知识：

层级 内容 示例（老子）
原始经典 人物原著的核心章节、名言、主题标签 "道可道非常道"、"上善若水"、"无为而无不为"
后人解读 现代学者/思想家的阐释和当代应用 陈鼓应《老子注译及评介》、安乐哲的过程哲学解读、韩林合的现代生存论解读

这样角色的回答不只是"老子说过什么"，还能吸收"当代人怎么理解老子"，让回复更丰富、更多层次。

2. 自由输入升级：自动在线研究

用户输入不在库中的名字时，系统自动执行三步：

Step 1 — 在线搜索
搜索 "[名字] 核心思想"、"[名字] 名言"、"[名字] 著作"，抓取 3-5 个来源的内容摘要

Step 2 — 提取角色资料
将搜索结果整理为：生卒年份、思想传统、代表著作、核心观点、语言风格、著名引文

Step 3 — 构建临时 System Prompt + 原典引用
将提取的资料拼入 System Prompt 和原典引用区，生成高质量角色回答
注意：在线检索仅在首次遇到该名字时执行。搜索结果缓存到本地，下次同一名字直接使用缓存。预设角色不需要在线搜索。

3. 两种路径对比

预设角色 自由输入
知识来源 手工整理的原典 + 后人解读 在线搜索 + LLM 知识
响应速度 即时（无网络 IO） 首次有 3-5 秒延迟，缓存后即时
质量 高（精细调校） 中高（取决于搜索质量）
角色数量 约 23 位 无限
