技术栈 & 部署

最终确认项

技术栈

层级 选型 理由
框架 Next.js 14 (App Router) 全栈一体，前端 + API 在一个项目里
数据库 SQLite (Prisma ORM) 零配置，适合小项目，足够用
认证 NextAuth.js 支持邮箱/GitHub登录，开箱即用
LLM OpenAI API (可替换) GPT-4o 或 Claude API 均可
搜索 SerpAPI / Bing API 用于自由输入时的在线检索
样式 Tailwind CSS 快速实现设计，与 Next.js 深度集成

部署

推荐 Vercel（Next.js 官方平台，免费额度够个人项目用），或自部署到任意 VPS。

SQLite 文件放在项目目录，简单直接。后续用户多了可平滑迁移到 PostgreSQL。

API 端点

方法 端点 功能
POST/api/question提交问题，返回三位思想家的首轮回答
POST/api/chat发送后续消息，继续一对一对话
GET/api/conversations获取用户的历史对话列表
GET/api/conversations/[id]获取单条对话完整记录
POST/api/share生成对话分享链接
GET/api/thinkers获取思想家列表（自动补全用）
