# 智者对话 — 设计文档

## 概述

一个面向个人与小圈子的 Web 应用。用户提问后，系统并行为三位不同思想传统的人物生成回复（宗教/哲学/社会学），用户选择一位展开多轮深度对话。回复以思想家口吻书写：先共情、再连接人类本源问题、最后引经据典。语言书面凝练，不回避沉重话题。

## 核心功能

1. **首轮三视角** — 用户提问后，三位思想家各回复 200-300 字精简回答，用户选择一个深入
2. **多轮对话** — 与选定思想家进行最多 15 轮对话（含首轮），结束后只读
3. **思想家选择** — 自由输入任意名字，支持自动补全；留空则系统自动从三种传统各选一位
4. **深度调节** — 温和模式（200-400字/轮）和深度模式（500-1000字/轮）
5. **对话历史** — 侧边栏展示历史对话列表，可回看
6. **分享链接** — 对话结束后生成只读分享页面

## 角色系统

### 预设角色配置结构

每个预设思想家包含以下字段：

| 字段 | 说明 |

### 自由输入处理

用户输入名字不在预设库中时：

1. 在线搜索该思想家资料（核心思想、名言、著作）
2. 提取构建临时角色所需字段
3. 生成临时 System Prompt
4. 搜索结果缓存到本地，后续使用缓存

## 回复规范

### 三段式结构（首轮回复）

1. **共情确认** — 一句话。识别并命名情绪，不急着引经据典
2. **连接本源** — 将个人困境与人类普遍处境相连，"你所面对的，与千百年来人类反复遭遇的...是一回事"
3. **经典点破** — 引用原文+白话解读，给出不粉饰的洞察

### 语言风格

- 书面凝练的现代汉语，不使用口语碎词
- 哲学/宗教概念使用术语本名，不稀释
- 禁用 LLM 式安全措辞（"作为AI"、"建议你注意情绪管理"）
- 允许深度讨论死亡、虚无、意义危机、社会批判等话题
- 后续轮次自然延续，不机械重复三段式

## 页面结构

### 路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | 提问表单，未登录用户唯一可见页面 |
| `/answers` | 三答页 | 三卡并排展示，选择一位深入 |
| `/chat/[id]` | 对话页 | 左侧历史列表 + 右侧对话区 |
| `/share/[id]` | 分享页 | 只读对话展示，无需登录 |

### 首页布局

- 居中大号文本输入框（用户问题）
- 思想家选择：自由文本输入框 + 自动补全，留空自动匹配
- 深度选择：温和 / 深度
- 提问按钮

### 三答页布局

- 三卡并排，每张卡片：头像 + 姓名 + 传统标签 + 200-300字回复 + "与此人深入对话"按钮
- 三条回答并行生成

### 对话页布局

- 左栏（240px）：历史对话列表，每条显示思想家、摘要、轮次计数、状态
- 右栏主区：对话气泡（用户右对齐、思想家左对齐）
- 底部输入区：文本输入 + 发送按钮 + 剩余轮次提示
- 顶部：当前思想家姓名、传统、轮次计数

---

## 对话规则

- 最大 15 轮（首轮三答计 1 轮，一对一深度对话 14 轮）
- 每轮保留完整上下文发送给 LLM
- 一轮对话锁定一位思想家，不可中途切换
- 用户输入限制 500 字
- 第 15 轮时系统提示"本轮对话即将结束"
- 结束后对话变为只读，显示"开启新对话"和"生成分享链接"按钮

## 技术栈

| 层级 | 选型 |
|------|------|
| 框架 | Next.js 14 (App Router) |
| 语言 | TypeScript |
| 样式 | Tailwind CSS |
| 数据库 | SQLite + Prisma ORM |
| 认证 | NextAuth.js |
| LLM | DeepSeek V4 API（兼容 OpenAI SDK） |
| 在线搜索 | SerpAPI / Bing Search API |
| 部署 | Vercel |

### 选择 DeepSeek V4 的理由

- 国内直连，支付宝/微信付款
- 中文原生写作质量高，对中国哲学传统理解更深
- API 兼容 OpenAI SDK，代码可随时切换
- 成本远低于 OpenAI/Claude（约 GPT-4o 的 1/5）
- 内容审查相对宽松，适合本项目需要的深度讨论

## API 端点

| 方法 | 端点 | 功能 |
|------|------|------|
| POST | `/api/question` | 提交问题，返回三位思想家的首轮回答 |
| POST | `/api/chat` | 发送后续消息，继续一对一对话 |
| GET | `/api/conversations` | 获取用户历史对话列表 |
| GET | `/api/conversations/[id]` | 获取单条对话完整记录 |
| POST | `/api/share` | 生成对话分享链接 |
| GET | `/api/thinkers` | 获取思想家列表（自动补全） |
| GET | `/api/thinkers/search?q=` | 在线搜索思想家资料（自由输入用） |

## 数据模型

```
User
  - id
  - email
  - name
  - createdAt

Conversation
  - id
  - userId
  - title (从首轮问题生成摘要)
  - thinkerName
  - thinkerTradition
  - isPresetThinker (预设/自由输入)
  - depthMode ("gentle" | "deep")
  - status ("active" | "completed")
  - currentRound (当前轮次)
  - shareId (分享链接标识，nullable)
  - createdAt
  - updatedAt

Message
  - id
  - conversationId
  - role ("user" | "thinker")
  - content
  - roundNumber
  - isFirstRound (是否首轮三答之一，用于标记被放弃的两个回复)
  - parentMessageId (首轮三答关联到用户问题)
  - createdAt

ThinkerConfig (预设角色配置)
  - id
  - name
  - tradition
  - era
  - voice
  - primarySources (JSON: string[])
  - secondarySources (JSON: string[])
  - systemPrompt
  - gentlePrompt
  - deepPrompt
  - avatar (emoji或首字母)
  - tags (JSON: 匹配用关键词)
```

## 自由输入在线检索流程

1. 用户输入名字 "维特根斯坦"
2. 系统检查 ThinkerConfig 表和本地缓存，未命中
3. 调用搜索 API（SerpAPI/Bing）搜索"维特根斯坦 核心思想"等
4. 提取关键信息构建临时角色对象
5. 生成适配本项目回复规范的临时 System Prompt
6. 缓存到本地数据库
7. 后续同一名字直接使用缓存
